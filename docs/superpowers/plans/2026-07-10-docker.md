# Docker Packaging (Step 4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package project_deal as a Docker image RLEaaS can build+run on any OS — one `docker build` bakes all 5 venvs (no Ray), one `docker run` serves the UI + reward doors on `$PORT`.

**Architecture:** Single-stage BuildKit Dockerfile. Wheels cached across builds via `uv` cache mounts; heavy deps installed before source (layer ordering) so rebuilds are fast. Secrets never baked — passed at runtime. The "test" is the real `docker build` succeeding (the from-scratch venv test) plus a `docker run` smoke (`GET /`, `GET /api/health`).

**Tech Stack:** Docker 29.0.0 (BuildKit), `python:3.12-slim`, `uv`, the project's existing venv recipe.

## Global Constraints

- **This is NOT pytest-TDD.** Verification = `docker build` (must exit 0) + `docker run` HTTP smoke. There is no "write a failing test first" — each task writes a file then runs its build/run verification.
- **Do NOT touch** any code: engine (`resources_server/`, `marketplace/`), `verifiers.py`, rubrics, `adapter.py`, `sim_ui/*.py`, `platform_export.py`. Step 4 only ADDS `Dockerfile`, `.dockerignore`, `.env.example` at the project_deal root.
- **Secrets never baked:** `.dockerignore` MUST exclude `.env`. Runtime keys via `docker run -e`. Cached UI needs NO key.
- **Pre-bake ALL 5 venvs at build, NO Ray:** engine `.venv`, `sim_ui/.venv`, + the 3 nemo_gym subserver venvs (`responses_api_models/openai_model`, `responses_api_agents/simple_agent`, `resources_servers/marketplace`). `skip_venv_if_present` makes ng_run skip venv setup on first boot.
- **Fast-rebuild caching:** BuildKit `--mount=type=cache,target=/root/.cache/uv` on every `uv` install; `ENV UV_CACHE_DIR=/root/.cache/uv`; deps-before-source layer ordering. **No `uv cache clean`** (the cache mount, not a layer, keeps the image lean while persisting wheels across builds). No multi-stage (editable `.pth` paths break across stages; size is a non-goal).
- **Image tag:** `project-deal-marketplace`. Build from `/home/azureuser/A2A_RL/project_deal`.
- **Env var names (exact):** `OPENROUTER_API_KEY`, `RLEAAS_CALLBACK_URL`, `RLEAAS_TOKEN`, `RLEAAS_ENV_NAME` (default `project_deal_marketplace`), `PORT` (default 8000).
- **No git commits.** Each task ends by logging a one-line note to `/home/azureuser/A2A_RL/CLAUDE.md`. Do NOT `git add`/`git commit`.
- **Paid live run inside the container is DEFERRED to the user** (boots ng_run + real rollout = OpenRouter credit). Do NOT run it.

### Task 2: `Dockerfile` + real build (the from-scratch venv test)

**Files:**
- Create: `/home/azureuser/A2A_RL/project_deal/Dockerfile`

**Interfaces:**
- Consumes: `.dockerignore` (Task 1).
- Produces: an image `project-deal-marketplace` with all 5 venvs baked; `CMD` runs `sim_ui.serve` on `$PORT`. Later tasks run it.

- [ ] **Step 1: Write `Dockerfile`** (verbatim from the spec §3)

```dockerfile
# syntax=docker/dockerfile:1
FROM python:3.12-slim

# system deps ng_run needs at RUN time: pgrep/pkill (procps), lsof, curl, bash, git, ca-certificates
RUN apt-get update && apt-get install -y --no-install-recommends \
      lsof procps curl bash git ca-certificates && rm -rf /var/lib/apt/lists/*

# uv
RUN curl -LsSf https://astral.sh/uv/install.sh | sh
ENV PATH="/root/.local/bin:${PATH}"
ENV UV_CACHE_DIR=/root/.cache/uv

WORKDIR /app/project_deal

# ---- heavy, rarely-changing layer: engine 3rd-party deps BEFORE source ----
COPY pyproject.toml uv.lock ./
RUN --mount=type=cache,target=/root/.cache/uv \
      uv venv --python 3.12 .venv && uv sync --no-install-project

# ---- source (busts on code change; deps layer above stays cached) ----
COPY . .
RUN mkdir -p nemo_gym_lib/cache

# ---- finish the 5 venvs (wheels come from the cache mount = fast rebuild) ----
# 1. engine: install the project + editable nemo_gym
RUN --mount=type=cache,target=/root/.cache/uv \
      uv sync && VIRTUAL_ENV="$PWD/.venv" uv pip install -e ./nemo_gym_lib
# 2. sim_ui
RUN --mount=type=cache,target=/root/.cache/uv \
      uv venv sim_ui/.venv && VIRTUAL_ENV="$PWD/sim_ui/.venv" uv pip install -r sim_ui/requirements.txt
# 3-5. subservers
RUN --mount=type=cache,target=/root/.cache/uv \
      for d in responses_api_models/openai_model responses_api_agents/simple_agent resources_servers/marketplace; do \
        ( cd "nemo_gym_lib/$d" && uv venv --seed --python 3.12 .venv && \
          VIRTUAL_ENV="$PWD/.venv" uv pip install -r requirements.txt ) ; done

ENV PORT=8000
EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=5s --start-period=20s --retries=3 \
  CMD curl -f "http://127.0.0.1:${PORT}/api/health" || exit 1
CMD ["sh", "-c", "sim_ui/.venv/bin/python -m sim_ui.serve"]
```

- [ ] **Step 2: Run the real build (the from-scratch test)**

Run: `cd /home/azureuser/A2A_RL/project_deal && DOCKER_BUILDKIT=1 docker build -t project-deal-marketplace . 2>&1 | tail -40`
Expected: build completes, exit 0, final lines show the image tagged. This may take several minutes on the first build (downloads GBs of wheels).

- [ ] **Step 3: Apply documented fallbacks IF the build fails** (only if Step 2 errored — otherwise skip)

- **`uv` install / PATH failure** (`uv: not found`): replace the uv install line with `RUN pip install uv` and drop the `ENV PATH=...local/bin` line (pip puts `uv` on the default PATH).
- **A wheel fails to compile** (`error: command 'gcc' failed` / `Building wheel ... did not run successfully`): add `build-essential` to the apt line: `... lsof procps curl bash git ca-certificates build-essential ...`.
- **`uv sync --no-install-project` errors** (e.g. it still tries to build a path dep before source exists): remove the early `COPY pyproject.toml uv.lock` + that first `uv sync --no-install-project` RUN, and instead do the engine venv in ONE step after `COPY . .`:
  ```dockerfile
  RUN --mount=type=cache,target=/root/.cache/uv \
        uv venv --python 3.12 .venv && uv sync && VIRTUAL_ENV="$PWD/.venv" uv pip install -e ./nemo_gym_lib
  ```
  (This loses the deps-before-source layer split but keeps the cache mount — rebuilds are still fast from cached wheels.)
- After any fallback edit, re-run Step 2. Record which fallback (if any) was applied.
